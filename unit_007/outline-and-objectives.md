# Unit 7 - Files, Exceptions, Error Handling, and JSON

This unit contains the following lessons:

- Lesson 1: Working with Files - Reading and Writing
- Lesson 2: File Modes, Paths, and Best Practices
- Lesson 3: Introduction to Exceptions and Error Handling
- Lesson 4: The try, except, else, and finally Blocks
- Lesson 5: Handling Common Exceptions
- Lesson 6: Working with JSON Data
- Lesson 7: Putting It All Together - Robust File Operations

## Lesson 1 Objectives

By the end of the lesson, students will be able to:

- explain why working with files is an important skill in programming.
- open and read text files using the `open()` function and `read()`, `readline()`, and `readlines()` methods.
- write data to a new or existing text file using the `write()` method.
- properly close files using the `close()` method or a `with` statement.
- reflect on how God has given us His Word as a permanent record — written down so it can be read, preserved, and passed on to future generations (Deuteronomy 6:6-9; 2 Timothy 3:16-17).

## Lesson 2 Objectives

By the end of the lesson, students will be able to:

- use different file modes (`'r'`, `'w'`, `'a'`, `'r+'`).
- work with file paths and understand relative vs absolute paths.
- safely open and close files using the `with` statement (context manager).
- follow best practices for reading from and writing to files.
- recognize that just as careful handling of important documents preserves truth, careful file handling preserves data integrity.

## Lesson 3 Objectives

By the end of the lesson, students will be able to:

- explain what exceptions and errors are in Python.
- understand why programs crash and how error handling makes programs more robust.
- identify common runtime errors (e.g., `FileNotFoundError`, `ValueError`, `TypeError`).
- distinguish between syntax errors and runtime exceptions.
- reflect on how life is full of unexpected situations, yet God calls us to respond wisely and with trust rather than panic (James 1:2-4; Proverbs 3:5-6).

## Lesson 4 Objectives

By the end of the lesson, students will be able to:

- use `try` and `except` blocks to catch and handle exceptions gracefully.
- use the `else` and `finally` clauses with try-except blocks.
- write code that continues running even when errors occur.
- handle specific exceptions rather than using a bare `except:` clause.
- understand that just as God provides grace when we fail, good error handling gives our programs a second chance to respond well.

## Lesson 5 Objectives

By the end of the lesson, students will be able to:

- handle common file-related exceptions (`FileNotFoundError`, `PermissionError`).
- handle data-related exceptions (`ValueError`, `TypeError`).
- use multiple `except` blocks for different types of errors.
- log simple error messages to help with debugging.
- recognize that learning to handle errors well builds resilience — a quality God desires in His people (Romans 5:3-5).

## Lesson 6 Objectives

By the end of the lesson, students will be able to:

- explain what JSON is and why it is commonly used for data storage and exchange.
- read JSON data from a file using the `json` module (`json.load()`).
- write Python data to a JSON file using `json.dump()`.
- convert between Python data types and JSON format.
- understand when JSON is a good choice for saving structured data.
- reflect on how JSON organizes information in a clear, structured way — similar to how God’s truth is orderly and can be clearly communicated (1 Corinthians 14:40).

## Lesson 7 Objectives

By the end of the lesson, students will be able to:

- combine file operations, exception handling, and JSON in practical programs.
- build robust programs that can read, process, save, and recover from errors gracefully.
- apply best practices for safe file handling and error management.
- debug programs that involve files and external data.
- reflect on how writing reliable, error-resistant code reflects the biblical call to be faithful and diligent in all we do (Colossians 3:23; 1 Corinthians 4:2).
