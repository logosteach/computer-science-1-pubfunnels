Unit 5 - All About Functions

This unit contains the following lessons:

  • Lesson 1: Introduction to Functions
  • Lesson 2: Defining and Calling Functions
  • Lesson 3: Function Parameters and Arguments
  • Lesson 4: Return Values and Scope
  • Lesson 5: Advanced Function Features
  • Lesson 6: Organizing Code with Functions
  • Lesson 7: Putting It All Together - Building Useful Programs

Lesson 1 Objectives

By the end of the lesson, students will be able to:

  • explain why functions are essential in programming and the principle of "Don't Repeat Yourself" (DRY).
  • describe how functions improve code readability, reusability, and maintainability.
  • identify real-world examples of functions (recipes, instructions, routines).
  • reflect on how God has designed the world and the Church with purpose and order, where different parts work together efficiently, just as functions help our code work together (1 Corinthians 12:12-27; Ecclesiastes 4:9-12).

Lesson 2 Objectives

By the end of the lesson, students will be able to:

  • define a simple function using the def keyword.
  • call (invoke) a function correctly.
  • understand the difference between defining and calling a function.
  • write functions that perform basic tasks and display output.
  • recognize that just as God calls us by name and gives us purpose, we “call” functions to perform specific tasks in our programs.

Lesson 3 Objectives

By the end of the lesson, students will be able to:

  • define functions with parameters.
  • pass arguments to functions using positional and keyword arguments.
  • use default parameter values.
  • understand how arguments are passed into functions.
  • reflect on how God communicates with us through His Word — clear instructions given with purpose, much like well-designed function parameters.

Lesson 4 Objectives

By the end of the lesson, students will be able to:

  • use the return statement to send values back from a function.
  • understand variable scope (local vs global variables).
  • explain what happens to variables inside and outside of functions.
  • avoid common scope-related errors.
  • reflect on how functions can “return” something valuable, just as our lives are meant to return glory and honor back to God (Matthew 5:16; 1 Corinthians 10:31).

Lesson 5 Objectives

By the end of the lesson, students will be able to:

  • use *args and **kwargs to accept a variable number of arguments.
  • write simple lambda functions for short, anonymous tasks.
  • understand when these advanced features are useful.
  • use them appropriately without making code overly complex.
  • recognize that God gives different gifts and levels of responsibility to different people, just as functions can flexibly handle varying inputs (Romans 12:6-8).

Lesson 6 Objectives

By the end of the lesson, students will be able to:

  • write clear docstrings to document their functions.
  • follow best practices for naming and structuring functions.
  • break large problems into smaller, reusable functions.
  • organize code into well-designed, readable functions.
  • reflect on how writing good functions teaches diligence and intentionality, qualities God desires in our work and service (Colossians 3:23; Proverbs 22:29).

Lesson 7 Objectives

By the end of the lesson, students will be able to:

  • combine functions with loops, conditionals, and data structures.
  • build small but useful programs using multiple functions.
  • plan and structure a program by breaking it into functions first.
  • debug programs that use multiple functions.
  • reflect on how well-organized functions reflect the biblical principle of doing all things “decently and in order” (1 Corinthians 14:40).
