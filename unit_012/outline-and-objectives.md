# Unit 12 - Windows Applications and GUIs with tkinter

This unit contains the following lessons:

- Lesson 1: Introduction to Graphical User Interfaces (GUIs)
- Lesson 2: Creating Your First tkinter Window
- Lesson 3: Adding Widgets - Labels, Buttons, and Entry Fields
- Lesson 4: Organizing Layout with Frames and Geometry Managers
- Lesson 5: Handling Events and User Interactions
- Lesson 6: Building a Simple tkinter Application
- Lesson 7: Putting It All Together - Creating Useful GUI Tools

## Lesson 1 Objectives

By the end of the lesson, students will be able to:

- explain what a Graphical User Interface (GUI) is and why it is important.
- understand the difference between console-based programs and GUI applications.
- recognize that tkinter is Python’s built-in library for creating desktop applications.
- reflect on how God has given humans the desire to create tools that are both useful and pleasant to use, reflecting His own creativity and care for beauty (Exodus 31:1-5; Psalm 90:17).

## Lesson 2 Objectives

By the end of the lesson, students will be able to:

- import the tkinter module and create a basic window.
- set the title, size, and basic properties of a tkinter window.
- run a tkinter event loop using `mainloop()`.
- understand the structure of a simple tkinter program.
- create and display their very first GUI window.

## Lesson 3 Objectives

By the end of the lesson, students will be able to:

- add and configure common tkinter widgets: `Label`, `Button`, and `Entry`.
- change text, colors, fonts, and other properties of widgets.
- retrieve text entered by the user from an `Entry` widget.
- connect a button to a simple function using the `command` parameter.
- begin building interactive elements in their GUI applications.

## Lesson 4 Objectives

By the end of the lesson, students will be able to:

- use the three main geometry managers: `pack()`, `grid()`, and `place()`.
- organize widgets neatly using `grid()` for table-like layouts.
- use `Frame` widgets to group and organize related components.
- create clean, professional-looking layouts for their applications.
- understand best practices for arranging GUI elements.

## Lesson 5 Objectives

By the end of the lesson, students will be able to:

- handle user events such as button clicks and text entry.
- write event handler functions that respond to user actions.
- update widgets dynamically based on user input.
- understand the event-driven programming model used in GUIs.
- create simple interactive behavior in their tkinter applications.

## Lesson 6 Objectives

By the end of the lesson, students will be able to:

- combine multiple widgets, layout techniques, and event handling.
- build a small, functional tkinter application (e.g., calculator, note taker, or converter).
- apply good coding practices when working with GUI code.
- debug common tkinter issues such as unresponsive windows or layout problems.

## Lesson 7 Objectives

By the end of the lesson, students will be able to:

- design and build a complete, useful desktop application using tkinter.
- combine knowledge from previous units (functions, conditionals, data structures, file handling) with GUI programming.
- create applications that are both functional and user-friendly.
- reflect on how creating tools that serve others reflects the heart of Christ, who came not to be served but to serve (Mark 10:45; Philippians 2:3-8).
