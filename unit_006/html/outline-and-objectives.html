Unit 6 - Object Oriented Programming and Classes

This unit contains the following lessons:

  • Lesson 1: Introduction to Object-Oriented Programming
  • Lesson 2: Creating Classes and Objects (Instances)
  • Lesson 3: Attributes, the __init__ Method, and Encapsulation
  • Lesson 4: Methods in Classes
  • Lesson 5: Inheritance and Code Reuse
  • Lesson 6: Polymorphism and Abstraction
  • Lesson 7: Putting It All Together - Building with Classes

Lesson 1 Objectives

By the end of the lesson, students will be able to:

  • explain what Object-Oriented Programming (OOP) is and its main benefits (organization, reusability, modeling real-world things).
  • describe the core ideas: classes, objects/instances, attributes, and methods.
  • understand how OOP helps manage complexity in larger programs.
  • reflect on how God created the world with distinct kinds of things, each with purpose and design — similar to how we use classes as blueprints (Genesis 1:11-12, 21, 24-25).

Lesson 2 Objectives

By the end of the lesson, students will be able to:

  • define a simple class using the class keyword.
  • create objects (instances) from a class.
  • understand the difference between a class (blueprint) and an instance (actual object).
  • recognize that God is the ultimate Creator who designs unique things, just as we create classes that produce many instances.

Lesson 3 Objectives

By the end of the lesson, students will be able to:

  • define and initialize attributes using the __init__ method and the self parameter.
  • access and modify instance attributes using dot notation.
  • explain the concept of encapsulation — bundling data and methods together while controlling access.
  • understand why encapsulation helps protect data and make code safer.
  • reflect on how God has given each person unique attributes and boundaries, just as encapsulation protects the internal state of an object (Psalm 139:13-14).

Lesson 4 Objectives

By the end of the lesson, students will be able to:

  • define and call methods inside a class.
  • use self to work with attributes within methods.
  • understand how methods give objects behavior.
  • write classes that contain both data (attributes) and actions (methods).
  • recognize that God created living things not only with attributes but also with the ability to act and respond.

Lesson 5 Objectives

By the end of the lesson, students will be able to:

  • explain inheritance and create a child class that inherits from a parent class.
  • override methods in the child class and extend parent functionality.
  • use the super() function when needed.
  • understand how inheritance promotes code reuse and reduces repetition.
  • reflect on how God’s blessings and calling can be passed down through generations while still allowing each person a unique relationship with Him (2 Timothy 1:5; Psalm 78:4-7).

Lesson 6 Objectives

By the end of the lesson, students will be able to:

  • explain polymorphism — the ability for different objects to respond to the same method name in their own way.
  • demonstrate polymorphism through method overriding and duck typing.
  • understand abstraction — hiding complex internal details and exposing only what is necessary.
  • use simple abstraction techniques (e.g., clear method names and avoiding direct attribute access when possible).
  • recognize that polymorphism and abstraction reflect how God works through many different people and situations while accomplishing the same good purposes (Romans 12:4-6; 1 Corinthians 12:4-7).

Lesson 7 Objectives

By the end of the lesson, students will be able to:

  • design and build small programs that use multiple classes, inheritance, and polymorphism.
  • apply encapsulation and abstraction appropriately in their code.
  • combine OOP with previous concepts (functions, loops, conditionals, lists, etc.).
  • debug and improve object-oriented code.
  • reflect on how well-designed classes reflect God’s character as the ultimate Designer — orderly, purposeful, and relational (Colossians 1:16-17; Romans 1:20).
