# Unit 10 - Testing and Debugging

This unit contains the following lessons:

- Lesson 1: The assert Statement
- Lesson 2: The `unittest` Module
- Lesson 3: The `pytest` Module

## Lesson 1 Objectives

By the end of the lesson, students will be able to:

- explain why testing is an essential part of professional programming.
- use the `assert` statement to check conditions during development.
- understand how `assert` statements can help catch bugs early.
- write simple assertions to validate function outputs and program state.
- reflect on how the practice of testing reflects the biblical call to “test everything” and hold on to what is good (1 Thessalonians 5:21).

## Lesson 2 Objectives

By the end of the lesson, students will be able to:

- explain the purpose and benefits of automated unit testing.
- write test cases using Python’s built-in `unittest` module.
- create test classes that inherit from `unittest.TestCase`.
- use common assertion methods such as `assertEqual()`, `assertTrue()`, and `assertRaises()`.
- run tests and interpret test results (passed, failed, or error).
- recognize that just as we are called to examine ourselves regularly (2 Corinthians 13:5), good developers regularly examine their code through testing.

## Lesson 3 Objectives

By the end of the lesson, students will be able to:

- explain the advantages of `pytest` over the built-in `unittest` module.
- install and use the `pytest` testing framework.
- write simple and readable test functions using `pytest`.
- use `pytest` fixtures for setup and teardown.
- run tests with different options and interpret detailed test output.
- understand when to use `assert`, `unittest`, and `pytest` in real projects.
- reflect on how thorough testing leads to more reliable and trustworthy programs, just as a life tested and refined by God becomes more fruitful and dependable (James 1:2-4; Psalm 26:2).
