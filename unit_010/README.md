# Unit 10 - Testing Your Code

This unit contains the following lessons: 

- The `assert` statement
- `unittest`
- `pytest`

## Lesson 1 Objectives

By the end of the lesson the student will..

- be able to define the purpose and usage of the Python `assert` keyword.
- be able to give examples of what the Python `assert` keyword is useful for and also when not to use the `assert` keyword.
- be able to identify when a block of code using an `assert` statement will throw an `AssertionError`
- be able to write code using comparison assertions
- be able to write code using membership assertions
- be able to write code using identity assertions
- be able to write code using assertions as documentation items
- be able to test their own programs with assertions

## Lesson 2 Objectives

By the end of the lesson the student will..

- be able to import the `unittest` module into their programs
- be able to write a `TestClass` that is a child class of `unittest.TestCase`
- be able to write `test_..` methods in the `TestClass`
- be able to write `test_..` methods that implement the assertion methods from `unittest.TestCase`
- be able to run a test using `unittest` from both the console/terminal and from withing a block of Python code.

## Lesson 3 Objectives

By the end of the lesson the student will..

- be able to download and install the `pytest` third party package indpendently into a virtual environment.
- be able to write `test_..` methods for testing with `pytest` using Python's `assert` keyword.
- be able to describe the advantages of using `pytest` over `unittest`.
- be able to describe the basic idea behind _boilerplate_ code.
- be able to write and implement the `@pytest.fixture` decorator in their programs.
- be able to write and implement the `@pytest.mark.parametrize` decorator in their programs.
- be able to write parametrizations of their test functions.
