Lesson 004 - Loop Control Statements: Break and Continue — Talking Points

Key talking points and discussion questions for teaching break and continue.

Main Talking Points

  • break statement: Immediately exits the current loop. Useful when you’ve found what you’re looking for or a condition is met.

  • continue statement: Skips the rest of the current iteration and moves to the next one. Useful for skipping unwanted values.

  • Practical Uses: - Searching for an item in a list (use break when found)
    - Skipping invalid input (use continue)

  • Important Rule: Both break and continue only affect the innermost loop in nested loops.

  • Common Beginner Mistake: Using break or continue without a clear purpose, making code harder to follow.

Discussion & Thought Questions

  • What is the difference between break and continue? When would you use one instead of the other?

  • Look at the search example. Why is using break more efficient than letting the loop finish?

  • In the invalid input example, why is continue useful? What would happen if we used break instead?

  • Can you think of a real-life situation where you might want to “break out” of a routine early?

  • Why is it important to understand how break and continue affect loop execution?

  • How does the concept of break (stopping wrong behavior) relate to the idea of repentance in the Bible?

  • How does continue (keeping going forward) relate to pressing on toward the goal in Christ (Philippians 3:13-14)?

Biblical Integration

In programming, break is like repentance — we stop going in the wrong direction. Continue is like pressing forward — we keep going toward the goal despite setbacks.
